# Icon Pack Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace all emoji characters in the Angular frontend with inline SVG icons from Phosphor Icons (bold weight), delivered through a shared `IconComponent`.

**Architecture:** A Node extraction script reads SVG files from the `@phosphor-icons/core` devDependency and writes `icons.constants.ts`. A tiny `IconComponent` renders those SVGs via `[innerHTML]`. Consumer files import `IconComponent` and swap emoji for `<app-icon>` elements or remove decorative emoji from button labels.

**Tech Stack:** Angular 17 standalone components, Phosphor Icons bold SVGs (MIT), Node.js script for generation.

## Global Constraints

- No new runtime npm dependencies — `@phosphor-icons/core` is devDependency only, used at generation time
- All CSS in `styles: []` arrays — no external stylesheets
- Angular 17 standalone components — no NgModules, no `@angular/animations`
- Template `*ngIf` / `@if` mix preserved as-is — do not convert existing directives
- Do NOT commit — user handles all git commits
- ASCII marks `✦`, `✓`, `✕` are not emoji — leave them unchanged

- [ ] **Step 2: Create the extraction script**

Create `frontend/scripts/extract-icons.mjs` with this exact content:

```javascript
import { readFileSync, writeFileSync, mkdirSync } from 'fs';
import { join, dirname } from 'path';
import { fileURLToPath } from 'url';

const __dir = dirname(fileURLToPath(import.meta.url));
const boldDir = join(__dir, '../node_modules/@phosphor-icons/core/assets/bold');
const outDir  = join(__dir, '../src/app/shared/constants');
const outFile = join(outDir, 'icons.constants.ts');

const ICON_MAP = {
  house:          'house-bold.svg',
  trophy:         'trophy-bold.svg',
  medal:          'medal-bold.svg',
  user:           'user-bold.svg',
  crown:          'crown-bold.svg',
  flame:          'flame-bold.svg',
  coin:           'coin-bold.svg',
  package:        'package-bold.svg',
  lightning:      'lightning-bold.svg',
  star:           'star-bold.svg',
  'person-run':   'person-simple-run-bold.svg',
  'person-walk':  'person-simple-walk-bold.svg',
  bicycle:        'bicycle-bold.svg',
  waves:          'waves-bold.svg',
  barbell:        'barbell-bold.svg',
  sneaker:        'sneaker-move-bold.svg',
};

const entries = Object.entries(ICON_MAP).map(([alias, file]) => {
  let svg = readFileSync(join(boldDir, file), 'utf-8').trim();
  // Normalise to 1em so CSS font-size controls the rendered size
  svg = svg.replace(/width="[^"]*"/, 'width="1em"')
            .replace(/height="[^"]*"/, 'height="1em"');
  return `  '${alias}': \`${svg}\``;
});

mkdirSync(outDir, { recursive: true });
const output =
  `// Auto-generated — do not edit by hand. Re-run scripts/extract-icons.mjs to regenerate.\n` +
  `export const ICONS: Record<string, string> = {\n${entries.join(',\n')},\n};\n`;

writeFileSync(outFile, output, 'utf-8');
console.log('Generated', outFile, `(${entries.length} icons)`);
```

- [ ] **Step 1: Create `icon.component.ts`**

Create `frontend/src/app/shared/components/icon/icon.component.ts`:

```typescript
import { Component, Input } from '@angular/core';
import { CommonModule } from '@angular/common';
import { ICONS } from '../../constants/icons.constants';

@Component({
  selector: 'app-icon',
  standalone: true,
  imports: [CommonModule],
  styles: [`:host { display: inline-flex; align-items: center; justify-content: center; line-height: 0; }`],
  template: `<span [innerHTML]="svg" [style.font-size.px]="size" style="display:inline-flex;align-items:center;line-height:0;"></span>`,
})
export class IconComponent {
  @Input({ required: true }) name = '';
  @Input() size: number = 20;
  get svg(): string { return ICONS[this.name] ?? ''; }
}
```

The `font-size` approach works because the extracted SVGs use `width="1em" height="1em"` — so setting `font-size: 20px` on the parent span makes the SVG render at 20×20px.

## Notes for Implementer

- **Icon sizing:** `<app-icon [size]="20" />` sets `font-size: 20px` on an inline span. The SVG uses `width="1em" height="1em"` so it renders exactly 20×20px. Do not set `width`/`height` on `<app-icon>` directly — use `[size]`.
- **Unknown icon name:** If `ICONS[name]` is undefined, `IconComponent.svg` returns `''` — the element renders empty. If you see a blank where an icon should be, the name string doesn't match any key in `icons.constants.ts`.
- **Regenerating icons:** If you need to add more icons later, add the alias → filename mapping to `extract-icons.mjs` and re-run `node scripts/extract-icons.mjs` from the `frontend/` directory.
- **`SPORT_ICONS` removal:** Only `dashboard.component.ts` imported `SPORT_ICONS`. No other file uses it, so removing it from `sport.constants.ts` won't break anything else.
- **Do not touch** `achievements.component.ts` or `profile.component.ts` — their `✦` and `✓` characters are ASCII decorative marks, not emoji.
